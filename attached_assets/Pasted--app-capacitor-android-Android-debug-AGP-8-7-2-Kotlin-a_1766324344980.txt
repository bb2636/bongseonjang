이 에러 메시지는 :app 모듈이 :capacitor-android 모듈을 Android 라이브러리(debug, AGP 8.7.2, Kotlin androidJvm)에 맞춰서 사용하려고 하는데, :capacitor-android 모듈 쪽에서 그에 해당하는 빌드 버전/변형(variant) 을 내보내지 못해서 의존성 해석에 실패했다는 뜻이다.
​

에러가 의미하는 것
:app 모듈에 implementation project(":capacitor-android") 같은 의존성이 걸려 있다.
​

Android Gradle Plugin(AGP) 8.7.2에서는 :capacitor-android가 다음 조건을 갖춘 Android 라이브러리 모듈이어야 한다.
​

debug 빌드 타입.

AGP 8용 속성 AgpVersionAttr = 8.7.2.

Kotlin 플랫폼 타입 androidJvm.

그런데 “No matching variant of project :capacitor-android was found” 라는 말은 :capacitor-android 모듈이:

아예 Android 라이브러리로 설정되지 않았거나,

AGP 8 기준으로 필요한 속성/설정이 잘못되어서 사용할 수 있는 variant가 없다는 뜻이다.
​

Capacitor 프로젝트에서 자주 나오는 원인
AGP 8에서 필수인 namespace 누락

AGP 8부터는 AndroidManifest.xml의 package만으로는 안 되고, 각 모듈의 build.gradle 안에 namespace를 명시해야 한다.
​

이게 없으면 AGP가 라이브러리 variant를 제대로 만들지 못해서 Gradle이 “matching variant 없음” 오류를 낸다.
​

낡은 Capacitor / 플러그인 + 최신 AGP 조합

예전 Capacitor 버전이나 플러그인들은 AGP 8 이전 기준으로 만들어져서 namespace나 Android 플러그인 설정이 최신 방식이 아닐 수 있다.
​

이런 상태에서 Gradle/AGP를 8.x로 올리면 variant 생성이 깨질 수 있다.

모듈이 Android 라이브러리로 설정되지 않음

:capacitor-android 모듈에는 반드시 com.android.library와 org.jetbrains.kotlin.android 플러그인이 적용되어 있어야 Android 라이브러리 variant가 생성된다.
​

실제로 확인하고 고칠 부분
android/settings.gradle 에 정의된 :capacitor-android 모듈이 가리키는 build.gradle 파일(보통 android/capacitor/build.gradle 같은 경로)을 열어서 아래를 확인하면 된다.

Android 라이브러리 + Kotlin 플러그인 적용 여부

모듈 build.gradle의 맨 위에 예를 들어 다음과 같은 내용이 있어야 한다.
​

text
plugins {
    id 'com.android.library'
    id 'org.jetbrains.kotlin.android'
}
Android 플러그인이 없거나 잘못된 플러그인을 쓰고 있다면, 위처럼 수정해서 AGP가 Android 라이브러리 variant를 만들 수 있게 해야 한다.

namespace 포함한 android 블록 추가

같은 파일 안에 android 블록과 namespace가 있어야 한다.
​

text
android {
    namespace "com.getcapacitor.android"   // 실제 패키지명에 맞게 수정
    compileSdkVersion 34                   // 예시
    defaultConfig {
        minSdkVersion 22
        targetSdkVersion 34
    }
}
AGP 8에서는 모든 Capacitor/플러그인 모듈이 이런 식으로 build.gradle 에 namespace를 직접 선언해야 한다.
​

모듈 포함 설정 확인

android/settings.gradle 에는 보통 다음과 비슷한 코드가 있다.
​

text
include ':app', ':capacitor-android'
project(':capacitor-android').projectDir = new File('../node_modules/@capacitor/android/capacitor')
경로가 잘못되어 폴더가 없거나 비어 있으면 Gradle 입장에서는 “내용 없는 프로젝트”가 되어서 variant가 생성되지 않는다.
​

Capacitor 버전과 AGP 버전 맞추기

최근에 Gradle/AGP를 8.x로 올렸다면, 사용하는 Capacitor 버전과 플러그인들도 AGP 8을 지원하는 버전으로 업데이트해야 한다.
​

당장은 업데이트가 어렵다면, 프로젝트 루트의 상위 build.gradle에서 AGP 버전을 Capacitor가 공식적으로 지원하는 7.x 등으로 낮춰서 빌드하는 방법도 있다.

다시 동기화 후 빌드

웹 프로젝트 루트에서 다음을 실행한다.
​

bash
rm -rf node_modules
npm install    # 또는 yarn / pnpm
npx cap sync android
그런 다음 android/ 폴더를 Android Studio로 열고 “Sync Project with Gradle Files” 를 실행한 후 다시 빌드해 본다.
