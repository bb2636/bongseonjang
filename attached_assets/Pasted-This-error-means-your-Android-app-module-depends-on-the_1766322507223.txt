This error means your Android app module depends on the :capacitor-android Gradle project, but that project does not expose any build variants that match what the Android Gradle Plugin 8.7.2 expects (debug, androidJvm, etc.), so Gradle cannot resolve it for compilation.

What the message is saying
project :app has a dependency implementation project(":capacitor-android").

Gradle looks for a matching variant of :capacitor-android with attributes:

AGP version 8.7.2

Build type debug

Kotlin platform androidJvm

Gradle reports No variants exist, which means the :capacitor-android module is not configured as a proper Android library for AGP 8+ (no android { ... } library config, or wrong plugin, or mis‚Äësynced project).

Most common root causes
Check these first in your project:

:capacitor-android is missing or broken

The android/capacitor folder might be missing, incomplete, or not properly linked as a Gradle module.
‚Äã

Wrong or outdated Capacitor / Android Gradle Plugin combination

Older Capacitor versions are not compatible with AGP 8.x and Gradle 8.x.
‚Äã

Missing namespace in module build.gradle

For AGP 8+, each Android module must define android { namespace "com.getcapacitor.android" } instead of relying on package in AndroidManifest.xml.
‚Äã

Bad Gradle or Kotlin plugin configuration

If the :capacitor-android module is not applying com.android.library and org.jetbrains.kotlin.android, it will not produce Android library variants.
